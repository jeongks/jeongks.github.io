Form Validation

Required information Format Data length Browser and web server will check data that meets the constraints of input data format. Client-sdie validation : done in the browser Server-side validation: validation done on the server.

Types of client-side validation

Built-in form validation uses HTML5 form validation feature. It dose not require JavaScript. It performs better than JavaScript, but it is not customizable. JavaScript validation is using JavaScript. Completely customizable.

Built-in form valication

Required - need to be filled to submit form Minlength & maxlength – specify length of input data(strings) Min & max - specify value of numberical input types Type – type of data Pattern – regular expression :valid pseudo-class & :invalid pseudo-class exist. Able to style it.

Regular Expression

a — Matches one character that is a (not b, not aa, and so on). abc — Matches a, followed by b, followed by c. ab?c—Matches a, optionally followed by a single b, followed by c. ( ac or abc) ab*c—Matches a, optionally followed by any number of bs, followed by c. ( ac , abc, abbbbbc, and so on). a|b — Matches one character that is a or b. abc|xyz — Matches exactly abc or exactly xyz (but not abcxyz or a or y, and so on).

Constraint validation API

Constraint Validation API
set of methods and properties available on the following form element DOM interfaces
HTMLButtonElement (represents a <button> element)
HTMLFieldSetElement (represents a <fieldset> element)
HTMLInputElement (represents an <input> element)
HTMLOutputElement (represents an <output> element)
HTMLSelectElement (represents a <select> element)
HTMLTextAreaElement (represents a <textarea> element)

Properties in Constraint validation API:
validationMessage: returns a localized message describing the validation constraints.
Validity: return a validityState.
